# Product API

A minimal Express + Sequelize REST API for managing products.

---

## 🚀 Quick start

1. Install dependencies:

   ```bash
   npm install
   ```

2. Configure environment variables in `.env` (see table below).

3. Start the app:

   ```bash
   npm start
   ```

   The server listens on `process.env.PORT` (defaults to `3000`). If you prefer another port, add `PORT=8080` (or whatever you want) to `.env`.

---

## ⚙️ Environment variables

Use `.env` to configure the app. The code expects these variables (defaults shown):

- `PORT` — server port (defaults to `3000`)
- `DB_DIALECT` — database dialect (`sqlite` | `mysql` | ...). Default: `sqlite`
- `DB_NAME` — database name (used for some dialects)
- `DB_USER` — DB username
- `DB_PASS` — DB password
- `DB_HOST` — DB host
- `DB_PORT` — DB port
- `DB_STORAGE` — sqlite file path (e.g., `./data/product.sqlite`)

> Note: Your `.env` currently contains `APP_PORT` and `DB_DIALET` — the app reads `PORT` and `DB_DIALECT`, so update those keys to avoid mismatches.

## 🔌 API Endpoints

Base path: `/api/products`

- GET `/api/products` — list all products
- GET `/api/products/:id` — get one product
- POST `/api/products` — create a product
- PUT `/api/products/:id` — update a product
- DELETE `/api/products/:id` — delete a product

Example `POST /api/products` payload:

```json
{
  "image": "https://example.com/image.png",
  "name": "Widget",
  "rating": { "avg": 4.5, "count": 10 },
  "priceCents": 1999,
  "keywords": ["gadget", "tool"]
}
```

- `keywords` is stored internally as a CSV string; you can send it as an array — the model setter will convert it.
- `id` is a UUID and is auto-generated by the model.

cURL example:

```bash
curl -X POST http://localhost:3000/api/products \
  -H "Content-Type: application/json" \
  -d '{"image":"https://example.com/image.png","name":"Widget","rating":{"avg":4.5,"count":10},"priceCents":1999,"keywords":["gadget","tool"]}'
```

## 🗄️ Database & schema notes

- The app uses Sequelize and calls `sequelize.sync({ alter: true })` on start. This will try to adjust tables to model changes.
- **Caution:** Adding `NOT NULL` columns to an existing table with rows will fail on SQLite (error: "Cannot add a NOT NULL column with default value NULL").

How to handle schema changes safely:

- Development (quick, destructive): delete the sqlite DB file (default `data/product.sqlite`) and restart — tables are recreated from models. This will remove existing data.
- Non-destructive (recommended when you want to preserve data): make new fields nullable or provide a `defaultValue` in the model so `ALTER TABLE` can succeed, or use migrations to add columns and then backfill values.
- Production: use a proper migration workflow (e.g., Sequelize migrations or a migration tool like Umzug/BFG) rather than `sync({ alter: true })`.

---

## ⚠️ Common gotchas

- If you change a primary key type (e.g., integer -> UUID), Sequelize's `sync({ alter: true })` may not handle it safely; use migrations.
- Getter for `keywords` may throw if the value is `null` — the model should guard for null values (e.g., return an empty array when null).
- `DataTypes.JSON` behavior differs across dialects; in sqlite it may be stored as TEXT.
